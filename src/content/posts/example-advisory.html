Example Advisory: SSRF via Unvalidated Webhook URL

This is a placeholder advisory post kept as a reference fixture. It demonstrates the advisory layout: a severity badge, an optional CVE tag, and the standard body.

Summary

A webhook configuration field accepted an arbitrary URL and the server fetched it without validating the host, allowing requests to internal-only services from an unauthenticated context.

Impact

An attacker could reach internal metadata endpoints and cloud credential services, escalating from unauthenticated access to sensitive infrastructure.

Remediation

Validate and allowlist outbound webhook hosts, block link-local and private ranges, and require authentication on the configuration endpoint.
